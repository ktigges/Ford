Ford Lightning EV Tool - Current Ops Notes (updated 2026-05-08)

Canonical references
- Schema source of truth: schema.sql
- Runtime config: config.json
- Install/start automation: scripts/install.sh, scripts/start_db_container.sh, start.sh
- Backup/restore automation: scripts/backup_db.sh, scripts/restore_db.sh
- Fresh restore verification: scripts/verify_fresh_install_restore.sh

OAuth auth init URI template
https://api.vehicle.ford.com/fcon-public/v1/auth/init?client_id=<CLIENT_ID>&state=<STATE>&redirect_uri=<REDIRECT_URI>

Current database shape
- PostgreSQL schema currently defines 22 tables.
- Charging model includes:
  - charging_state (latest snapshot)
  - charging_history (time-series samples)
  - charging_sessions (persistent session lifecycle)
- Drive model includes:
  - drives
  - drive_points
- schema.sql includes: CREATE EXTENSION IF NOT EXISTS pgcrypto;

Recommended install on a new server
1) Clone and enter repo
   git clone <repo>
   cd Ford

2) First-time DB install + tuning
   ./scripts/install.sh

3) Start app
  ./start.sh

One-command deploy wrapper
- Standard deploy:
  ./scripts/deploy.sh
- Deploy including restore:
  ./scripts/deploy.sh --restore-sql backups/<backup_file>.sql

Database persistence and restart behavior
- DB container name: lightning-db
- DB volume name: lightning_pgdata
- Data persists in Docker volume across container restarts/removals.
- Subsequent restarts should use:
  - ./scripts/start_db_container.sh (DB only)
  - ./start.sh (DB + app)
- These restart scripts do not create a new container.
- Poller should be started/stopped from the app UI (`/poller`) or app setting (`autostart_poller`).

Migration/backup workflow
- Create migration backup:
  ./scripts/backup_db.sh migration

- Restore on destination host:
  ./scripts/install.sh --restore-sql backups/<backup_file>.sql

- Validate restore in isolated fresh DB:
  ./scripts/verify_fresh_install_restore.sh backups/<backup_file>.sql

Charging data reset
- Script: scripts/reset_charging_data.sql
- Clears charging_history, charging_sessions, charging_state
- Resets charging_history.id and charging_sessions.id sequences when target_vin is NULL

Operational note
- Keep secret.key secure and backed up if encrypted oauth client secrets must remain decryptable on same host.
- JSON backup restores re-encrypt oauth client_secret with destination host key.
